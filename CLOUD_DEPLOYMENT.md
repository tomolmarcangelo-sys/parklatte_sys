# Cloud Deployment Guide: Aiven MySQL + Render

This guide explains how to connect your app to **Aiven MySQL** and deploy it to **Render**.

## Step 1: Set up Aiven MySQL
1.  **Create an Account**: Go to [aiven.io](https://aiven.io/) and sign up.
2.  **Create a Project**: Create a new project (e.g., `coffee-shop-system`).
3.  **Create MySQL Service**:
    *   Select **MySQL**.
    *   Choose a Cloud Provider (e.g., Google Cloud or AWS) and a region closest to you.
    *   Select the **Free Plan** (if available) or the smallest Hobbyist plan.
4.  **Wait for Initialization**: Once status is "Running", look for the **Connection Information**.
5.  **Get Credentials**:
    *   **Host**: `mysql-xxxxx-project.aivencloud.com`
    *   **Port**: `xxxxx` (usually not 3306)
    *   **User**: `avnadmin`
    *   **Password**: *Generated by Aiven*
    *   **Database**: `defaultdb` (or create one named `coffee_shop`).
6.  **Run migrations**: Use a tool like **DBeaver** or **MySQL Workbench** to connect to Aiven and run the SQL from `db.sql`.

## Step 2: Prepare for Render
1.  **Push to GitHub**: Make sure all your code (including the new `server.ts`) is pushed to a private GitHub repository.
2.  **Ensure package.json is correct**:
    ```json
    "scripts": {
      "dev": "tsx server.ts",
      "build": "vite build",
      "start": "NODE_ENV=production node server.ts"
    }
    ```

## Step 3: Deploy to Render
1.  **Log in to Render**: Go to [render.com](https://render.com/).
2.  **Create a New Web Service**:
    *   Connect your GitHub account.
    *   Select the repository for this project.
3.  **Configure Environment**:
    *   **Name**: `coffee-shop-system`
    *   **Region**: Same as your Aiven DB.
    *   **Runtime**: `Node`
    *   **Build Command**: `npm install && npm run build`
    *   **Start Command**: `npm start`
4.  **Add Environment Variables**:
    Go to the **Environment** tab in Render and add:
    *   `DB_HOST`: *Your Aiven Host*
    *   `DB_PORT`: *Your Aiven Port*
    *   `DB_USER`: `avnadmin`
    *   **DB_PASSWORD**: *Your Aiven Password*
    *   `DB_NAME`: `defaultdb` (or your custom name)
    *   `DB_SSL`: `true`
    *   `JWT_SECRET`: *A long random string (e.g., click "Generate" or use `openssl rand -base64 32`)*
    *   `NODE_ENV`: `production`

## Step 4: Finalize
Once the build is complete on Render, your app will be live!
*   **Real-time updates**: Your app uses Socket.io, which Render supports automatically on standard Web Services.
*   **Static Assets**: Vite will build the frontend, and the production server will serve them from the `dist` folder.

## Troubleshooting SSL
If you see "ACCESS DENIED" or SSL errors:
1. Ensure `DB_SSL=true` in Render.
2. If Aiven requires a specific CA cert, you can download the `ca.pem` from Aiven, add it to your project, and point to it in `server.ts`. However, `rejectUnauthorized: false` usually works for most cloud-managed MySQL instances.
